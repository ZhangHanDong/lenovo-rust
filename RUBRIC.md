# 附录 B · 评估维度与"作业—报告"机制

## 作业机制

L1–L15 每课一道评分作业（AI Coding 任务卡），经 **Git PR 提交**（贴近真实评审流程，并支撑"代码审查"训练）。每道作业按下述统一维度打分（每维 **0–3**：未涉及 / 初步 / 熟练 / 精通），未触及的维度记"未涉及"。

> **V3 特有的评分项**：每个 PR 必须回答三个 AI 审查问题（见文末 **D8**）——**这是本课程与其他 Rust 课程最大的不同**。

## 八个能力维度（D1–D8）

| 能力维度 | 评分要点 | 主要相关课 |
|---|---|---|
| **D1 正确性** | 功能满足规格、测试通过 | 全部 |
| **D2 所有权与借用** | 无不必要 clone、借用与生命周期建模合理、零拷贝权衡 | L2、L3 |
| **D3 类型与错误处理** | 以类型表达状态（非法状态不可表达）、错误分层、`Option` / `Result` / `?` 得当 | L4、L5、L8 |
| **D4 并发、异步与性能** | 无数据竞争、`.await` 点与真实并发、`select!` / 优雅关闭、先测量再优化 | L6、L7、L8 |
| **D5 unsafe / FFI 边界** | `unsafe` 契约清晰、`// SAFETY:` 诚实、**miri 全绿**、为 C/C++ 构建安全封装、跨界类型正确 | L9、L10 |
| **D6 跨平台 / Windows** | `windows` crate 集成、句柄 RAII、COM/服务 typestate、cargo-xwin/MSI/CI、洋葱架构（core 零 `#[cfg]`） | L11–L14 |
| **D7 可审查性 / 工程质量** | clippy 干净、可读、符合规格、提交规范、回归测试 | 全部 |
| **D8 AI 协作判断力** | 能指出 AI 生成代码的错误、能说出否决 AI 建议的技术理由、用编译器 / miri / criterion 验证而非盲信 | **全部（V3 核心）** |

## 评分锚点（每维 0–3 的判据）

| 分值 | 含义 | 判据示例（以 D2 为例） |
|---|---|---|
| 0 | 未涉及 | 作业未触及所有权设计 |
| 1 | 初步 | 能通过编译，但存在不必要 clone / 过度 `Arc` |
| 2 | 熟练 | 所有权建模合理，借用清晰，无冗余拷贝 |
| 3 | 精通 | 在零拷贝与可维护性间做出有理据的权衡 |

**D8（AI 判断力）的评分锚点**：

| 分值 | 含义 | 判据 |
|---|---|---|
| 0 | 未涉及 | PR 没回答 AI 三问，或只写"AI 帮我写的" |
| 1 | 初步 | 能说 AI 帮了什么，但没指出任何错误 |
| 2 | 熟练 | 指出了 AI 的至少一处问题，并说明怎么发现的 |
| 3 | 精通 | 指出 AI 的错误 + 用工具（编译器/miri/criterion）验证 + 说清正确解法的技术理由 |

## PR 必答三问（D8 的落地）

**每个 AI Coding 作业的 PR 描述里，必须回答**：

1. **AI 帮了什么？**（哪些代码是它写的、省了多少时间）
2. **你否决了 AI 的哪些建议？为什么？**（要有具体技术理由，不是"我觉得"）
3. **AI 犯了什么错？你是怎么发现的？**（编译器抓的 / miri 抓的 / criterion 抓的 / 你自己看出来的）

> 各课的 AI 典型错误清单见教材各课"AI Coding 实战"小节——例如 L3 的"修借用报错就加 clone"、L9 的"写出有 UB 的 unsafe"。

## 报告生成机制

- **个人学习评估报告（每位学员）** = 八个维度的当前水平 + 跨课次趋势 + 作业完成 / 通过率 + 结业考核得分 + 强弱项与下一步学习建议。
- **团队能力报告（整班）** = 各维度得分分布 + 团队最薄弱维度（能力缺口）+ 对企业优先领域（**Windows 跨平台 / C/C++ 互操作 / AI 协作审查**）的就绪度评估与后续发展建议。

### 个人报告模板（示意）

```text
学员：____________
─────────────────────────────────────
维度         L1  …  L15    结业   趋势
D1 正确性     2   …   3       3     ↗
D2 所有权     1   …   3       3     ↗↗
D3 类型/错误  2   …   3       2     →
D4 并发/异步/性能 - …  2       2     ↗
D5 unsafe/FFI -   …   2       2     新增
D6 跨平台/Win -   …   3       3     新增
D7 工程质量   2   …   3       3     ↗
D8 AI 判断力   1   …   3       3     ↗↗
─────────────────────────────────────
强项：所有权建模、Windows 跨平台集成
弱项：异步背压处理
建议：复盘 L7 select! 与优雅关闭，重做 WinMon 采集器的取消路径
```

> 因评分维度统一、可机器辅助汇总，覆盖 50–100 人时边际成本可控。

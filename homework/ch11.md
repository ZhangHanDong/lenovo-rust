# 第 11 课作业 · Windows 目标与打包链路

## 作业目标

把“能编译”扩展到目标信息、静态运行时、签名步骤和 CI 说明，建立 Windows 分发链路意识。

## 代码位置

| 用途 | 路径 |
|---|---|
| 交叉编译与打包 | `projects/v2-win-packaging` |
| 目标配置 | `.cargo/config.toml` |
| CI 示例 | `ci/ci.yml` 或项目内工作流文件 |

## 需求

1. 增加 CLI `--target-info` 或同等 subcommand，打印 target 相关信息。
2. 输出至少包含 `target_pointer_width`、`target_env` 和是否 MSVC。
3. 增加至少 1 个平台无关测试。
4. 在 `.cargo/config.toml` 中为 `x86_64-pc-windows-msvc` 配置 `+crt-static`。
5. 补全 signtool 示例步骤，使用占位证书并说明 timestamp `/tr`。

## PR 说明必须包含

- 静态 CRT 解决什么分发问题；
- 它带来的体积、更新或兼容性取舍；
- 哪些验证可以在 macOS / Linux 做，哪些必须在 Windows runner 或实机做。

## 验证命令

```bash
cargo test -p v2-win-packaging
cargo clippy -p v2-win-packaging --all-targets -- -D warnings
cargo fmt -p v2-win-packaging --check
cargo xwin build --release -p v2-win-packaging --target x86_64-pc-windows-msvc
```

## 自检清单

- [ ] `--target-info` 输出可被测试稳定判断。
- [ ] signtool 步骤不包含真实证书或密码。
- [ ] PR 中清楚区分构建、安装、运行和签名验证。

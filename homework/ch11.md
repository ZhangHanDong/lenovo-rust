# L11 作业 · 再封装一个 Win32 API

> 对应教材 L11「Win32 与 windows-rs」。代码骨架在 `stages/L11-win32/`。

## 背景

采集器要加"内存/磁盘信息"。真实 Win32 调用在 `#[cfg(windows)]` 里（见 SOLUTION，需 Windows 机器跑）；骨架里平台无关可测的是 `format_bytes` 与 RAII `HandleGuard`。

## 需求

封装 `GlobalMemoryStatusEx`（内存）**或** `GetDiskFreeSpaceExW`（磁盘），二选一。补全骨架里 `format_bytes` 与 `HandleGuard::drop` 的 `todo!()`，Windows 封装照 SOLUTION 在 Windows 上完成。

## 验收标准

- **在 Windows 机器上实际跑通**（截图/输出贴 PR）；
- 所有句柄用 RAII 守卫管理，**代码里零个手写 `CloseHandle`**；
- 所有 Win32 调用错误用 `?` 传播（不许 `unwrap`）；
- 需要 `dwSize` 的结构体都填了；
- 有一个**错误路径测试**（如无效盘符返回 `Err` 而非崩溃）；
- **PR 里回答**（D8 评分项）：
  1. 这个 API 有几条失败路径？每条路径上句柄会被正确关闭吗？
  2. AI 的第一版，四项检查过了几项？

> 非 Windows 机器上这段在 `#[cfg(windows)]` 里不参与编译；用 `cargo check --target x86_64-pc-windows-msvc`（装 `cargo-xwin` 后）在 Mac 上做类型检查。

# 第 6 课作业 · RingBuffer 安全抽象

## 作业目标

在含 `unsafe` 的数据结构中新增能力，同时保持 drop、迭代和内部不变量正确。

## 代码位置

| 用途 | 路径 |
|---|---|
| unsafe 与安全抽象 | `projects/v2-safe-abstraction` |

## 需求

1. 为 `RingBuffer<T>` 增加 `push_overwrite(&mut self, value: T) -> Option<T>`。
2. 缓冲区未满时行为等同普通 push，返回 `None`。
3. 缓冲区已满时覆盖最旧元素，返回 `Some(old)`。
4. 保持 `head`、`len` 和初始化槽位不变量正确。
5. 实现 consuming `IntoIterator` 或 borrowed `iter()` 二选一。
6. 增加 Drop 计数测试，证明没有泄漏或 double free。

## PR 说明必须包含

- 每个 `unsafe` 块对应的不变量；
- 覆盖元素时旧值和新值的 drop 责任；
- 为什么公开 API 仍然是安全的。

## 验证命令

```bash
cargo fmt -p v2-safe-abstraction --check
cargo clippy -p v2-safe-abstraction --all-targets -- -D warnings
cargo test -p v2-safe-abstraction
cargo +nightly miri test -p v2-safe-abstraction
```

- [ ] 覆盖满缓冲区时返回的是最旧元素。
- [ ] 迭代顺序与逻辑顺序一致。
- [ ] miri 能通过新增测试。

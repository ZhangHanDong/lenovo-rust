# 第 8 课作业 · Flutter Rust Bridge 类型映射

## 作业目标

在 flutter_rust_bridge 工程中新增可调用 Rust API，并确认 Dart 侧生成类型符合预期。

## 代码位置

| 用途 | 路径 |
|---|---|
| Flutter 主线工程 | `frb_demo/` 的副本 |
| Rust API 文件 | `rust/src/api/simple.rs` |

## 需求

1. 复制一份 `frb_demo/` 作为作业工程，不直接修改原始演示工程。
2. 在 `rust/src/api/simple.rs` 中新增 `fn parse_amount(s: String) -> Result<i64, String>`。
3. 再新增一个返回 `Option<String>` 的函数。
4. 运行 `flutter_rust_bridge_codegen generate` 重新生成绑定。
5. 确认 Dart 绑定中出现 `Future<int>` 和 `String?`。
6. 在 `test/rust_call_test.dart` 中增加至少 2 个测试：正常解析和解析失败异常。

## PR 说明必须包含

- `Result<i64, String>` 在 Dart 侧如何表现；
- 为什么失败不应该用 `-1` 这类哨兵值表达；
- 哪类函数适合标注 `#[frb(sync)]`。

## 验证命令

在作业工程内运行：

```bash
flutter test test/rust_call_test.dart
```

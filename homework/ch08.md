# L8 作业 · 让 v3-core 能上生产

> 对应教材 L8「工程化：错误分层、测试与性能素养」。代码骨架在 `stages/L8-robust/`。

## 需求（三件事，缺一不可）

1. **消灭所有 `unwrap`**（测试除外）：库层用 `thiserror` 定义结构化错误，应用层用 `anyhow` + `.context()` 传播；坏数据**跳过但计数**，不静默。
2. **测试覆盖**：单元测试每个解析分支 + 集成测试完整链路 + 至少一个 `compile_fail` 测试（保护类型约束）。
3. **用 criterion 证明一处优化有效**：先测量 → 找热点 → 改一处 → 再测量（按 `SOLUTION.md` 加 `benches/`）。

骨架给出 `ParseError` / `parse_line` / `collect_valid` / `require_healthy`，补全 `todo!()`。

## 验收标准

- `cargo clippy -- -D warnings` 全绿；
- `grep -rn "unwrap()" src/ | grep -v test` 结果为空；
- **PR 里回答**（D8 评分项）：
  1. 你优化了什么？**criterion 说快了百分之多少？**
  2. AI 建议的优化里，**有哪一条被数据证明是无效的？**

## AI 在这一课的典型错误

- `unwrap()` 图省事（说"生产记得改"——没人会改）；
- `Box<dyn Error>` 一把梭（丢掉错误类型信息）；
- 吞错误：`let _ = do_something();`（最危险——错误消失，bug 还在）；
- 凭直觉优化，且说得极其自信。

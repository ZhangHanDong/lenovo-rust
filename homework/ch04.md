# 第 4 课作业 · Tokio 协议与关闭通知

## 作业目标

扩展异步协议，练习共享计数、广播通知和 cancel-safety 的基本判断。

## 代码位置

| 用途 | 路径 |
|---|---|
| Tokio 异步服务 | `projects/v2-async` |

## 需求

1. 增加 `COUNT` 协议命令，返回已经处理的行数。
2. 使用 `Arc<AtomicUsize>` 维护计数，并传入 `handle_conn`。
3. 保持 `handle_line` 为纯函数，不在其中绑定连接状态。
4. 使用 `broadcast` 在关闭前通知所有已连接客户端。
5. 增加至少 2 个确定性测试：`COUNT` 端到端测试、广播通知接收测试。
6. 测试不能依赖 `sleep`。

## PR 说明必须包含

- 是否存在跨 `.await` 持有非 `Send` 值的风险；
- 是否存在同步阻塞调用卡住 runtime 的风险；
- 为什么计数放在连接外部，而不是每个连接单独保存。

## 验证命令

```bash
cargo test -p v2-async
cargo clippy -p v2-async --all-targets -- -D warnings
cargo fmt -p v2-async --check
```

## 自检清单

- [ ] `COUNT` 返回的是服务级处理行数，而不是单连接行数。
- [ ] 关闭广播能被多个客户端观察到。
- [ ] 异步测试有明确同步点。

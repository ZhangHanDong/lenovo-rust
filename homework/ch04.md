# L4 作业 · 事件模型建模

> 对应教材 L4「类型系统 I：用类型表达约束」。代码骨架在 `stages/L4-model/`。

## 背景

WinMon 要处理日志事件。一条事件可能是：**进程启动**（进程 ID + 进程名）、**进程退出**（进程 ID + 退出码）、**资源告警**（资源类型 + 百分比）、**心跳**（无数据）。

## 需求

设计 `Event` 类型 + `describe(e: &Event) -> String`（骨架给出类型脚手架，补全 `describe` 的 `todo!()`）。

## 协作方式（别让 AI 先污染你的思路）

1. **先自己在纸上设计**——这是关键；
2. 让 AI 也设计一版；
3. **对比**：AI 的版本能构造出多少种非法状态？你的呢？
4. 提交更好的那版（可能是你的，也可能是融合版）。

## 验收标准

- `ProcessId` 用 newtype，不是裸 `u32`（防止和退出码混用）；
- **零个非法状态**——每个变体只带它真正需要的数据；
- `describe` 的 `match` **不写 `_` 兜底**；
- 有一个测试确认：给 `Event` 加新变体时，所有 `match` 都编译失败（**把"演化提醒"变成可验证资产**）；
- **PR 里写明：AI 的建模能构造多少种非法状态，你的是多少**（D8 评分项）。

## AI 在这一课的典型错误

- 用 `String` 当类型标签（`kind: String`——魔法字符串，拼错编译器也不知道）；
- 所有字段塞进一个 struct，用 `Option` 表示"这个变体没这字段"；
- `match` 写 `_ => {}` 兜底（放弃了演化提醒）；
- 用裸 `u32` 表示 ID（`ProcessId` 和 `ExitCode` 都是整数，能互相传参，编译器不拦你）。

# 第 14 课作业 · 调试、tracing 与回归测试

## 作业目标

从一个可复现缺陷出发，完成定位、日志证据、修复和回归测试闭环。

## 代码位置

| 用途 | 路径 |
|---|---|
| 调试技巧 | `projects/v2-debugging` |

## 需求

1. 增加一个带 `// BUG:` 标记的缺陷函数，例如错误的 `Vec::remove` 下标、累加溢出或过早 `break`。
2. 增加对应的 fixed 版本。
3. 添加 `tracing` instrumentation，能暴露关键状态。
4. 增加缺陷复现测试，可以是 `should_panic` 或明确的错误断言。
5. 增加修复后的回归测试。
6. 可选：使用 `cargo expand --lib` 查看 `#[instrument]` 或 derive 展开。

## PR 说明必须包含

- `RUST_LOG=trace` 输出片段；
- 哪一行日志暴露了缺陷；
- 为什么新增测试能防止同类回归。

## 验证命令

```bash
cargo test -p v2-debugging
cargo clippy -p v2-debugging --all-targets -- -D warnings
cargo fmt -p v2-debugging --check
RUST_BACKTRACE=1 cargo test -p v2-debugging buggy_window
```

## 自检清单

- [ ] 缺陷函数和修复函数的差异足够清楚。
- [ ] trace 输出能支持定位结论。
- [ ] 回归测试失败时能指向本次修复的核心行为。

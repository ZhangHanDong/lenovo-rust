# 第 7 课作业 · C ABI 与安全封装

## 作业目标

新增一个 C ABI 函数，并在 Rust 侧提供安全封装，重点关注跨语言整数宽度、指针和所有权。

## 代码位置

| 用途 | 路径 |
|---|---|
| FFI 与 C/C++ 互操作 | `projects/v2-ffi-cpp` |

## 需求

1. 在 C 侧新增 `int ffi_sum(const int* arr, size_t n, int64_t* out)`。
2. 空数组返回错误。
3. Rust 侧添加手写 `extern` 声明。
4. Rust 侧提供安全 API：`sum(&[i32]) -> Option<i64>`。
5. 增加至少 2 个测试，包括空切片。

## PR 说明必须包含

- 为什么输出使用 `int64_t`，而不是 `long`；
- `arr` 指针的生命周期和所有权归属；
- 如果 C 侧返回 malloc 字符串、Rust 侧直接释放，会有什么风险；
- 可选：说明何时应该引入 bindgen。

## 验证命令

```bash
cargo test -p v2-ffi-cpp
cargo clippy -p v2-ffi-cpp --all-targets -- -D warnings
cargo fmt -p v2-ffi-cpp --check
```

## 自检清单

- [ ] 空数组和非空数组路径都被测试覆盖。
- [ ] 安全封装不暴露裸指针给调用者。
- [ ] PR 中明确说明 Windows LLP64 下 `long` 的陷阱。

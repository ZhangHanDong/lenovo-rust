# 第 2 课作业 · 配置加载与错误分层

## 作业目标

扩展配置加载器，让错误类型、批量加载和借用边界更接近真实工程代码。

## 代码位置

| 用途 | 路径 |
|---|---|
| 所有权与错误处理 | `projects/v2-ownership-errors` |

## 需求

1. 为配置增加 `tls: bool` 字段，默认值为 `false`。
2. 增加 `name` 字段非空校验，非法时返回 `Invalid { field: "name", .. }`。
3. 实现 `load_many(&[&str]) -> (Vec<Config>, Vec<ConfigError>)`，允许部分成功。
4. 增加至少 2 个测试，覆盖新增字段、校验失败或批量加载。
5. 可选：实现 `host_and_retries_mut(&mut self) -> (&mut String, &mut u32)`，或在 PR 中复现并修复一次 E0502 借用错误。

## PR 说明必须包含

- 为什么库代码不直接使用 `anyhow` 作为公开错误类型；
- 哪些场景下 `Cow` 可以减少分配；
- 批量加载为什么返回成功列表和错误列表，而不是遇到第一个错误就退出。

## 验证命令

```bash
cargo fmt -p v2-ownership-errors --check
cargo clippy -p v2-ownership-errors --all-targets -- -D warnings
cargo test -p v2-ownership-errors
```

## 自检清单

- [ ] `name` 为空时错误字段名准确。
- [ ] `tls` 未配置时使用默认值。
- [ ] `load_many` 不会因为单个坏配置丢弃已成功解析的配置。

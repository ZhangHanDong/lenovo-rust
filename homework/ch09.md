# L9 作业 · RingBuffer 安全封装

> 对应教材 L9「Unsafe：信任边界与安全封装」。代码骨架在 `stages/L9-unsafe/`。

## 背景

高频采样需要一个定长环形缓冲——满了就覆盖最老的，创建后**零堆分配**。内部用 `MaybeUninit` + 裸操作，外部是安全 API。

## 需求

实现 `Ring<T>`，含 `with_capacity` / `push` / `get` / `len`。`with_capacity` / `len` / `Drop` 是讲师带敲的基础骨架（已给出），补全 `push` 和 `get` 的 `todo!()`。

## 协作方式

1. 讲师带敲基础骨架；
2. 让 AI 补全剩余方法；
3. **用 miri 验证**——AI 的版本能过吗？
4. 补测试，直到 miri 干净。

## 验收标准（**miri 是硬门槛**）

- `cargo test` 全绿；
- **`cargo +nightly miri test` 全绿**（CI 门禁）；
- 每个 `unsafe` 块都有 `// SAFETY:` 注释，且注释里每条都能被代码验证；
- **测试覆盖每一个 unsafe 分支**（miri 看不到没跑过的代码）；
- **PR 里写明**（D8 评分项）：
  1. 这个类型的**不变量**是什么？
  2. 每个 unsafe 块依赖哪一条不变量？谁保证它成立？
  3. **miri 抓到了 AI 的什么错误？**（一个都没抓到 = 测试覆盖不够，回去补）

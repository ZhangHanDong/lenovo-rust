# 第 9 课作业 · Win32 API 安全封装

## 作业目标

从一个 Win32 API 场景开始，建立 `windows` crate feature、句柄 RAII 和平台无关返回值的基本写法。

## 代码位置

| 用途 | 路径 |
|---|---|
| Win32 API | `projects/v2-win-api` |

## 需求

在下面两个方向中选择一个完成。

## 方向 A：读取注册表

1. 启用 `Win32_System_Registry`。
2. 读取 `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProductName`。
3. 对外返回平台中立的 `Option<String>`。
4. 非 Windows 平台提供 stub。
5. 使用 RAII 管理 `HKEY`。

## 方向 B：枚举顶层窗口

1. 使用 `EnumWindows` 和 `GetWindowTextW`。
2. 返回 `Vec<WindowInfo { hwnd_id, title }>`。
3. 正确处理 callback、UTF-16 字符串和空标题。

## 通用要求

1. 增加至少 2 个可在非 Windows 桌面运行的平台无关测试。
2. 精确列出新增 `windows` feature。

## PR 说明必须包含

- 句柄、字符串和 HRESULT 的 soundness 边界；
- 为什么公开 API 不直接暴露 Win32 原始类型；
- feature 列表为什么是最小集合。

## 验证命令

```bash
cargo test -p v2-win-api
cargo clippy -p v2-win-api --all-targets -- -D warnings
cargo fmt -p v2-win-api --check
```

- [ ] 非 Windows 平台可以编译并运行平台无关测试。
- [ ] Windows 专有代码都在 `cfg(windows)` 下。
- [ ] 所有 Win32 资源释放路径明确。

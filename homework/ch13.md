# 第 13 课作业 · 跨平台核心与 CI

## 作业目标

把平台差异收束到 trait 和 adapter 内，让核心逻辑可以用 mock 测试，并在三平台 CI 中验证。

## 代码位置

| 用途 | 路径 |
|---|---|
| 跨平台核心 | `projects/v2-cross-core` |
| CI | `.github/workflows/` |

## 需求

1. 为 `PlatformInfo` 增加 `temp_dir() -> PathBuf`。
2. Windows 使用 `%TEMP%` 语义，Unix 使用 `$TMPDIR`，缺省回退到 `/tmp`。
3. 所有平台实现都要补齐。
4. 增加核心函数 `cache_file_path(&dyn PlatformInfo, app, file) -> PathBuf`。
5. 增加至少 2 个 mock 注入测试。
6. 增加三平台 CI matrix workflow，可以只运行本 crate。

## PR 说明必须包含

- `cfg` 覆盖是否存在重叠或空洞；
- 为什么路径拼接放在核心函数中，而平台差异放在 adapter 中；
- CI matrix 如何证明主要平台没有编译断层。

## 验证命令

```bash
cargo test -p v2-cross-core
cargo clippy -p v2-cross-core --all-targets -- -D warnings
cargo fmt -p v2-cross-core --check
```

## 自检清单

- [ ] mock 测试不依赖本机真实临时目录。
- [ ] Windows、macOS、Linux 的 `cfg` 分支互不冲突。
- [ ] workflow 只做本课需要的验证，不引入无关任务。

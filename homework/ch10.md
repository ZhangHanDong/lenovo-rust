# L10 作业 · 接入 C 校验库

> 对应教材 L10「FFI：与 C/C++ 互操作」。代码骨架在 `stages/L10-ffi/`（含 `csrc/checksum.c` 与 `build.rs`）。

## 背景

`v3-core` 要接入一个"遗留的" C 校验库（内置 `csrc/checksum.c`），事件入库前走它校验。

## 需求

1. 用 `cc` crate 编译 C 代码（`build.rs` 已给出）；
2. 声明 `extern "C"` 绑定（或用 bindgen 生成）；
3. **写安全封装层**——上层业务代码不碰任何 unsafe（补全 `checksum` / `describe` 的 `todo!()`）；
4. 处理好 `wm_describe()` 返回的 C 字符串（**谁分配谁释放**）。

## 验收标准

- `cargo test` 全绿，**含空切片、超长数据的边界测试**；
- 所有跨边界 struct 标 `#[repr(C)]`；每个 `unsafe` 块有 `// SAFETY:` 注释；
- **业务层代码里零 unsafe**；
- **PR 里回答**（D8 评分项）：
  1. `describe()` 返回的内存，**谁分配的？谁释放的？** 怎么保证不漏？
  2. 如果 C 库改成"返回的字符串由调用方 free"，你的封装要改哪里？
  3. AI 的第一版绑定里，**五项检查有几项没过？**

> miri 不支持真实 C 调用；改用 ASan 时，Rust 侧使用 `RUSTFLAGS="-Zsanitizer=address" cargo +nightly test`，C 侧也要以 `-fsanitize=address` 编译并链接兼容运行库。

# 第 10 课作业 · COM / WinRT 生命周期

## 作业目标

练习 COM / WinRT 的对象创建、引用计数或回调实现，并把平台绑定隔离在可审查边界内。

## 代码位置

| 用途 | 路径 |
|---|---|
| COM / WinRT | `projects/v2-win-com` |

## 需求

在下面两个方向中选择一个完成。

## 方向 A：WinRT consumer

1. 创建 `Windows::Foundation::Uri`。
2. 读取 `Domain()` 或 `SchemeName()`。
3. 把结果转换为项目内的 `CallRecord` 或等价结构。

## 方向 B：最小回调 / 事件接收器

1. 使用 `#[implement]` 实现一个最小 callback 或 event sink。
2. 让该 callback 被调用一次。
3. 记录调用结果，便于测试或日志验证。

## 通用要求

1. 增加至少 2 个可在非 Windows 桌面运行的平台无关测试，例如 HRESULT 分类或 `CallRecord` 格式化。
2. 精确列出新增 `windows` feature。

## PR 说明必须包含

- COM apartment 的初始化边界；
- 引用计数、`QueryInterface` 和 double release 风险；
- 为什么核心测试不依赖 Windows 实机。

## 验证命令

```bash
cargo test -p v2-win-com
cargo clippy -p v2-win-com --all-targets -- -D warnings
cargo fmt -p v2-win-com --check
```

## 自检清单

- [ ] Windows 绑定代码不泄漏到平台无关核心。
- [ ] PR 中说明对象生命周期由谁负责。
- [ ] feature 列表与使用的 API 对应。

# L12 作业 · WinMon 服务 + companion 异常通知

> 对应教材 L12「COM / WinRT 与系统集成」。代码骨架在 `stages/L12-service/`。

## 需求

1. **typestate 服务状态机**（`Service<Stopped/Running/Paused>`，骨架已给，补全转移方法的 `todo!()`）；
2. 服务安装/卸载（`sc create`/`sc delete` 或 `windows-service` crate，Windows 上完成）；
3. 服务采到异常事件（如某进程 CPU > 90%）时，经本机 IPC 发给登录会话中的 `WinMonTray` companion，由 companion **弹 toast 通知**；
4. 关键事件写**事件日志**。

## 验收标准

- **在 Windows 机器上实际安装并启动服务**（`sc query WinMon` 截图贴 PR）；
- **状态机是 typestate 的**——`compile_fail` 测试证明"在 Stopped 上调 resume 编译不过"；
- 服务回调里有 `catch_unwind`；配置文件用**绝对路径**；所有 COM 对象零手写 `AddRef`/`Release`；
- toast 代码不进入 Session 0 服务；IPC 有消息长度上限、本机限制和只允许预期账户访问的 ACL；
- **PR 里回答**（D8 评分项）：
  1. 你的状态机能表达多少种状态？其中有非法的吗？
  2. AI 的第一版用什么方式表达状态？它能构造非法状态吗？
  3. `compile_fail` 测试保护的是哪一条业务规则？
